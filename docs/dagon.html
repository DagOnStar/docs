class dagon.task.Task(name, command, working_dir=None, endpoint=None)[source]¶

Bases: threading.Thread

Represents a task executed by DagOn

It can be executed on local machine, cloud instance, cluster or in a docker container

Variables:
  • name (str) – unique name of the class
  • command (str) – command to be executed
  • working_dir (str) – path to the folder or directory where the task is going to be executed
  • nexts (list[]) – tasks with dependencies to this tasks, that will be executed when this task ends
  • prevs (list[]) – tasks that has to be executed before of this task (dependencies to be resolved)
  • reference_count (int) – number of references to this task
  • remove_scratch_dir (bool) – True if the sratch directory has to be removed after the execution of this task
  • running (bool) – True if the task is in execution
  • workflow (dagon.Workflow) – workflow related to this task
  • status (dagon.Status) – actual dagon.Status of the task
  • info (dict(str, object)) – information of the enviroment where this task is going to be executed
  • endpoint (str) – globus endpoint id (only used when dagon.DataMover is DataMover)
Parameters:
  • name (str) – name of the task
  • command (str) – command to be executed by the task
  • working_dir (str) – path to the directory where the task is be executed
  • endpoint (str) – globus Endpoint ID
