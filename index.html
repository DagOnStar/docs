class dagon.task.Task(name, command, working_dir=None, endpoint=None)[source]¶

Represents a task executed by DagOn

It can be executed on local machine, cloud instance, cluster or in a docker container

Variables:
  • name – unique name of the class
  • command – command to be executed
  • working_dir – path to the folder or directory where the task is going to be executed
  • nexts – tasks with dependencies to this tasks, that will be executed when this task ends
  • prevs – tasks that has to be executed before of this task (dependencies to be resolved)
  • reference_count – number of references to this task
  • remove_scratch_dir – True if the sratch directory has to be removed after the execution of this task
  • running – True if the task is in execution
  • workflow – workflow related to this task
  • status – actual dagon.Status of the task
  • info – information of the enviroment where this task is going to be executed
  • endpoint – globus endpoint id (only used when dagon.DataMover is DataMover)
Parameters:
  • name – Name of the task
  • command – Command to be executed by the task
  • working_dir – Path to the directory where the task is be executed
  • endpoint – Globus Endpoint ID

class dagon.task.Types[source]¶

Enum used to represent the main types tasks supported by dagon, remote tasks are supported by the RemoteClass but it should not be explicitly instantiate

Variables:
  • BATCH – Regular Batch task (local or remote)
  • SLURM – Task executed using Slurm (local or remote)
  • CLOUD – Task executed in a cloud instance (ec2, digital ocean and google cloud tested with libcloud)
  • DOCKER – Task executed on a docker container (local or remote)

Class that represents a task executed on a cloud instance

Variables:
  • instance_id (str) – ID of the instance given by the cloud provider
  • provider (libcloud.compute.types.Provider) – cloud provider (see libcloud.compute.types.Provider)
  • instance_flavour (dict) – characteristics of the instance
  • key_options (dict) – ssh key options to connect to the instance
  • node (libcloud.compute.base.Node) – Node where the task will be executed
  • stop_instance (bool) – True if the instance has to be stopped when the execution ends
  • instance_name (str) – Name of the instance on the cloud provider
Parameters:
  • name (str) – name of the task
  • command (str) – command to be executed on the cloud instance
  • provider (libcloud.compute.types.Provider) – cloud provider
  • ssh_username (str) – UNIX username to connect through SSH
  • key_options (dict) – dictionary with the key options to connect to the instance
  • instance_id (str) – instance id on the cloud provider
  • instance_flavour (dict) – characteristics of the instance (image, size, location)
  • instance_name (str) – instance name on the cloud provider
  • stop_instance (bool) – True if the instance has to be stoppend at the end of the task execution
  • endpoint (str) – Globus endpoint ID
execute()[source]¶

Execute the task on the remote cloud instance.

  1. Create or get the instace from the cloud provider.
  2. Get the public IPS available.
  3. Open the SSH connection with the instace.
  4. Execute the task.
